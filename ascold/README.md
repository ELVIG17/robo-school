npm run dev

## структура проекта

модульная архитектура проекта 

-- bash 
/src
--pages/     # страница регистрации/домашняя страница/страница товара/АДМИН ПАНЕЛЬ,  ТО ЧТО ОТОБРАЖАЕТСЯ В URL
--modules/   # модули со своей бизнес логикой
--components/  # менее самостоятельные куски кода карточки товара, не имеет запросов на сервер и бизне  логики 
--ui/ # самые простые переиспользуемфе компоненты

## Чего не должно быть в pages:
Собственные API-запросы
Сложная бизнес-логика
Состояние (кроме просто ui-состояния)
## характеристики  modules 
Имеет свою бизнес-логику
Работает с API
Имеет состояние local или global
Может содержать внутренние компоненты
Решает одну конкретную задачу
## теристики components
Переиспользуется в разных модулях/страницах
Нет собственных API-запросов
Нет сложной бизнес-логики
Может иметь простое состояние(isOpen, isActive) эти состояния не работают с сервером.
Получает данные через props
## как определить куда что класть
вопрос1: это точка входа роутера ДА > pages нет > идем дальше 
вопрос2: имеет сложную бизнес логику apl, СОСТОЯНИЕ да > MODULES НЕТ > ИДЕМ ДАЛЬШЕ 
ВОПРОС3: ЭТО БАЗОВЫЙ UI ЭЛЕМЕНТ (КНОПКА ИНПУТ ) ДА > ul no > components
